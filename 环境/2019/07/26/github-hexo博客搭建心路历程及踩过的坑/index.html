二、安装Hexo、博客构建及推送至github

只需简单的几个命令即可完成博客本地搭建及推送至github，首先在自己电脑中找个地方新建一个文件夹，作为你的本地博客目录，比如我在我的E盘下新建一个myBolg文件夹

  1. 安装Hexo
    在上面所建的文件夹下鼠标右键打开Git Bash，输入以下命令开始安装
    npm install -g hexo-cli

  1. 初始化

    hexo init [myBlog]
    注：myBlog这个目录我们早就建好了，若没有，系统会帮我们创建

    完成之后会生成以下文件和目录

  1. 安装npm
    npm install
    不是很懂这步做了什么，有啥用
    ![](/images/8.png)

  1. 本地预览
    hexo s

    在浏览器中输入localhost:4000，可以看到我们的博客已经生成了。

  1. 生成html静态文件，推送（部署）至github
    （1）生成静态文件

    `hexo g`

    生成之后，会发现本地博客目录中多了一个文件夹public和db.json数据文件，这个public就是我们要推送至github上的文件夹。

（2）找到并修改全局配置文件_config.yml，打开该文件，拉到最后。添加下图所示deploy（部署）代码
![](/images/3.png)

注：冒号后面需要留一个空格，否则不生效，这是我第一个踩过的坑，就因为没有留意这里要空格，浪费了时间重复做前面的步骤。

（3）在部署之前需要安装hexo-deployer-git插件，不然报以下错，这是第二个踩过的坑

`npm install hexo-deployer-git --save`

   ERROR Deployer not found: git

（4）部署

`hexo d`
![](/images/11.png)

从红色框框中可以看到，部署成功，现在在浏览器中输入https://你的用户名.github.io 就会看到你的博客啦！

部署在github的文件如下，其实就只有前面public文件夹下的东西，所以要是本地myBlog的博客目录丢失了，想通过git clone命令克隆到本地是不行的，因为public之外的东西没有了，所以只能重新在本地搭建环境并生成部署了，这是踩过的第三个坑，因为我就这么做过，之前电脑重装系统了，没备份，那些东西自然没有了。
